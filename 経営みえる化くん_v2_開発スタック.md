# 経営みえる化くん v2.0 - 開発スタック

## プロジェクト概要

小規模店舗（治療院）向けの予約管理・経営分析システム。日常の経理業務から経営分析、キャッシュフロー管理までを1つのWebアプリで完結。

## 3モード構成

### 1. 経理モード（💰）

- 予約一覧表示（フィルタ・検索・ソート）
- インライン編集（金額、ステータス、決済方法、メモ）
- 決済方法別集計サマリー
- 50:50レイアウト（リスト/詳細パネル）

### 2. 分析モード（📈）

- **ダッシュボード:** 今月売上、来院数、リピート率、離反予備軍/確定
- **顧客分析:** 来院回数分布、平均来院回数
- **売上トレンド:** 月別売上・来院数推移（12ヶ月）
- **メニュー分析:** メニュー別売上構成

### 3. CFモード（🧩）

- ブロックパズル図（売上→粗利→営業利益の可視化）
- 経費入力フォーム（7項目）
- リアルタイム計算（粗利率、労働分配率、税引後利益、繰越金）
- Expensesシートへ保存
- 前月コピー機能

## 実装順序（推奨）2025/12/2

1. **初期データ一括取得＋クライアントフィルタ化**  
   - `getInitialData`（フィルタ選択肢＋対象月の予約＋サマリー）を新設し、初回ロードを1リクエストに統合。  
   - 同一年月のフィルタ/検索/ソートはフロントで処理し、再リクエストを発生させない。
2. **Chart.js の遅延ロード＆軽量化**  
   - `<script>` に `defer` を付与、または分析モード遷移時に動的 import して初期表示を非ブロッキング化。  
   - グラフ再描画時はアニメーションを無効化し、既存インスタンスを再利用してちらつきを抑制。
3. **予約リスト描画の分割/バーチャル化**  
   - `DocumentFragment`＋`requestAnimationFrame` で50件単位など小分け描画に変更。  
   - スクロール領域内のみを描画する簡易バーチャルリスト化で大量件数でも滑らかに。
4. **GAS側キャッシュ・集計の再利用**  
   - `getAnalysisBundle`/`getMonthlySales` など年月単位の結果を `CacheService` で5–10分キャッシュ。  
   - 必要に応じて月次集計シートを作成し、分析系関数の重複計算を排除。
5. **UX/エラーハンドリングの強化**  
   - ローディングオーバーレイを `try/finally` で必ず閉じ、シート欠損や列不整合はユーザー向けメッセージを表示。  
   - 主要フローに軽量なスモークテスト（売上合計/リピート率/離反数）を追加し回帰検知を容易にする。

## 手動テスト（スモーク）

- モード切替と年月同期: ヘッダーの年月変更で経理/分析/CFすべてが同じ年月に同期し、分析KPIとCFブロックの売上が更新されること。
- 経理: 予約一覧が表示され、フィルタ・検索が反映される。予約を編集して保存後、サマリーが更新される。
- 分析: 分析タブを開くとKPI・売上グラフ・離反リストが表示される。同じ年月でタブを開き直しても正しく表示される。
- CF: CFタブで売上が自動反映され、経費入力変更でブロックと結果が即座に再計算される。保存ボタンでエラーなく完了する。
- 空/少データ時: Reservations/Patients/Expenses が空でもクラッシュせず、0表示で動作する。
